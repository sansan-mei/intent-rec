# README First, README Last

## 适用场景

当用户要求你进行任何代码相关操作时（包括但不限于：新增功能、修复 bug、重构、配置调整、脚本变更、依赖安装与升级），必须启用本流程。

## 强制规则

1. **先读 README，再写代码**

   - 在开始任何代码编辑前，先阅读仓库根目录 `README.md`。
   - 如果改动发生在某个子目录，继续阅读该目录的 README（例如 `plugin/README.md`）。
   - 未阅读 README 时，禁止开始编码。

2. **实现过程对齐 README 约定**

   - 代码实现要遵守 README 中已有的约定（目录结构、命令、开发规范、运行方式等）。
   - 若 README 与当前需求冲突，优先满足用户明确要求，并在最终文档更新中注明变更原因。

3. **写完代码必须更新 README**

   - 代码改动完成后，检查 README 是否需要同步更新。
   - 任何影响使用方式、配置、命令、行为、接口、限制条件的改动，都必须更新对应 README。
   - 若确认 README 无需修改，也要在回复中明确说明“README 已核对，无需更新”。

4. **交付前自检**
   - 确认“已读 README -> 已改代码 -> 已更新 README/已说明无需更新”三步全部完成。
   - 任一步未完成，不可视为任务完成。

## 执行清单（每次任务都要走）

- [ ] 阅读 `README.md`
- [ ] 阅读相关子目录 README（如存在）
- [ ] 实施代码改动
- [ ] 同步更新 README（或明确说明无需更新）
- [ ] 在最终回复中说明 README 处理结果
